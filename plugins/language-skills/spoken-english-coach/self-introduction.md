# 英文面试自我介绍

> 创建时间：2024-03-10
> 表达风格：口语化、自然

---

## 完整英文版本

Hello, nice to meet you. I'm very glad to have the opportunity to attend this interview. My name is Wang Jiasheng.

I graduated from Fudan University with a master's degree. Now I work at Trip.com as a senior front-end engineer.

I passed CET-6.

I've worked at two companies.

My first job was at a bank. I did some work on frontend infrastructure.

My primary work was to establish a ready-to-use frontend framework. It provides basic component libraries and CLI tools. CLI is a short term for Command Line Interface. It helps business teams quickly build admin systems.

My second job is at Trip.com. My work roughly has two directions.

One is business feature development.

I was mainly responsible for all development tasks of the Trip.com overseas activity detail page. This page supports three platforms: app, mobile web, and PC web.

I migrated the existing architecture to a new architecture called xTaro, which is Trip.com's new cross-platform framework. Write once, run on multiple platforms: app, H5, and PC.

For the PC version, my project was the first one to go live.

I did a lot of work on page load speed. The app launch rate improved from 61% to 94%.

On the web side, the Lighthouse score reached 60. This is based on real user data from different regions, so it's higher in Beijing but lower in Europe and the US.

Second Direction is that I built a frontend testing solution.

The core goal of this solution is to let test developers write unit test scripts by themselves without depending on the front-end engineer. It connects several internal platforms, like mock platform and testhub platform.

I was involved in the entire project lifecycle: initial research, core development, and team support.

I built mock environments for H5 and App. I also created shared assertion libraries and validation commands for common scenarios, like tracking verification, routing, and i18n. These cover over 80% of our test cases.

Now all our team's repos use this solution. We set up CICD gates too. Line coverage is over 90%, branch coverage is above 70%. We're also sharing it with other teams.

This solution was really valuable when we refactored our framework.

Third, I've been actively exploring AI tools recently. I'm very familiar with popular AI tools like Claude Code and Cursor. During the Spring Festival, I developed several AI-related tools, including VSCode extensions, npm packages, and Skills. The total downloads have been quite good.

In summary, my strength: I have both B2B and B2C experience. I've done well in performance optimization, quality assurance, and engineering. My biggest strength is using engineering methods to boost team efficiency.
