# 零干预边界模型（Autonomy Gate Model）

> Harness 的终极目标：让长任务在需求澄清后完全自主运行。

## 模型定义

```
  人类参与         │         AI 自主执行
 ──────────────── ┼ ───────────────────────────
  CLARIFY          │  SEARCH → SPEC → BLUEPRINT
  (说清楚要什么)    │  → TEST → IMPLEMENT → VERIFY
                   │  (自动规划 → 自动执行)
                   │
                   └──── 零干预边界（Autonomy Gate）
```

## 两个阶段

### 阶段一：人机协作（CLARIFY）

**人类职责**：
- 说清楚"做什么"
- 说清楚"不做什么"
- 确认"怎样算完成"
- 提供必要的背景信息

**AI 职责**：
- 通过提问确保理解完整
- 整理需求为结构化的四要素（模块/主题/边界/目标）
- 在理解不完整时主动追问

**退出条件**：四要素全部明确，人类确认需求清晰。

### 阶段二：AI 自主执行（SEARCH → VERIFY）

**AI 全权负责**：
- 自动规划：将需求分解为有序子任务
- 自动检索：查找已有实现，避免重复造轮子
- 自动规格化：生成 Spec 文件
- 自动蓝图：生成完整蓝图（受影响文件、依赖、风险、任务清单）
- 自动测试：先写测试，再写实现
- 自动执行：按清单逐项实施
- 自动验证：运行测试，确认完成

**人类角色**：
- 可随时查看进度文件（被动观测）
- 不需要主动参与任何决策
- VERIFY 完成后收到结果汇报

## 自主执行的四个能力

### 1. 自动任务分解

将一个长需求拆解为可执行的短任务序列：

```
长需求：实现用户认证功能

→ 分解为：
  1. 设计认证数据模型 [无依赖]
  2. 实现注册 API [依赖 1]
  3. 实现登录 API [依赖 1]
  4. 实现 JWT 中间件 [依赖 1]
  5. 保护路由端点 [依赖 2,3,4]
  6. 编写集成测试 [依赖 5]
```

**关键特征**：
- 分解结果有编号和依赖关系
- 分解结果持久化到文件
- 执行时按依赖顺序推进

### 2. 自主阶段推进

CLARIFY 之后的阶段间自动衔接：

```
SEARCH 完成 → 自动检查检索结果
  ├─ 找到可复用实现 → 自动进入 SPEC
  └─ 未找到 → 自动标注"从零开始" → 自动进入 SPEC

SPEC 完成 → 自动检查规格完整性
  ├─ 四要素完整 → 自动进入 BLUEPRINT
  └─ 不完整 → 自动回到 CLARIFY（自愈）

BLUEPRINT 完成 → 自动检查蓝图完整性
  ├─ 任务清单 + 风险清单齐全 → 自动进入 TEST
  └─ 不齐全 → 自动补充缺失项（自愈）
```

**关键特征**：
- 阶段间由条件自动触发，无需人工 approve
- 条件检查是机械的、可自动判定的
- 不是"跳过门禁"，而是"门禁自动化"

### 3. 自愈能力

遇到错误时自动诊断和处理：

| 错误类型 | 自愈策略 |
|----------|----------|
| 测试失败 | 自动分析失败原因，修正实现，重新运行 |
| 依赖缺失 | 自动安装依赖，重新尝试 |
| 阶段产出不完整 | 自动补充缺失项（如蓝图缺少风险清单则自动补充"无已知风险"） |
| 代码冲突 | 自动分析冲突，选择安全策略解决 |
| 多次重试失败 | 降级执行（简化流程绕过问题），在报告中标注 |

**关键特征**：
- 不阻塞、不等待人类介入
- 有回退（回到安全状态）和降级（简化执行）两层策略
- 所有自愈行为记录在状态文件中

### 4. 进度透明

执行过程中状态可观测但非阻塞：

```
.harness/
├── specs/             # 需求规格（人类可读）
├── blueprints/        # 蓝图文件（含任务状态 ⏳/✅/❌）
├── state.json         # 当前阶段 + 进度百分比
└── logs/              # 执行日志（自动追加）
```

**关键特征**：
- 状态实时写入文件，不依赖终端输出
- 人类可随时查看，但查看行为不影响执行流
- VERIFY 完成后主动汇报结果

## 评估标准

评估一个 Skill 的自主执行能力时，问以下问题：

1. **CLARIFY 之后，人类还需要做什么吗？**
   - 什么都不需要 → D7 高分
   - 只在关键节点确认 → D7 中等
   - 每步都要确认 → D7 低分

2. **如果 AI 遇到错误，会发生什么？**
   - 自动诊断 + 回退/降级 + 继续 → D7 高分
   - 重试几次然后阻塞 → D7 中等
   - 直接阻塞等待人类 → D7 低分

3. **一个长任务能跑多久不需要人看？**
   - 可以从头跑到尾 → D7 高分
   - 需要在蓝图确认时介入 → D7 中等
   - 每个阶段都要人 → D7 低分

## 反模式

### 反模式 1：伪自主（阶段间仍需人工 approve）

```
CLARIFY → [人类确认] → SEARCH → [人类确认] → SPEC → ...
```

这不是自主执行，只是把交互从"每步确认"变成了"每阶段确认"。

### 反模式 2：无自愈（出错即崩溃）

```
TEST → 测试失败 → 💥 阻塞，等待人类修复
```

没有自愈能力的"自主执行"只是"延迟崩溃"。

### 反模式 3：黑箱执行（无法观测进度）

```
CLARIFY → ... ... ... → 3小时后 → "做完了" 或 "出错了"
```

人类无法在执行过程中了解进度，只能等待最终结果。
