# 评分细则

## 等级定义

| 总分 | 等级 | 标签 | 含义 |
|------|------|------|------|
| 48-60 | S | Balanced Harness | 纪律与效率兼顾，流程自适应，AI 可控且高效 |
| 36-47 | A | Well Harnessed | 大部分到位，少量改进即可 |
| 24-35 | B | Partially Harnessed | 有基本控制，但关键门禁或适配能力缺失 |
| 12-23 | C | Loosely Harnessed | 约束不足或流程过重，AI 容易失控或效率低 |
| 0-11 | D | Unharnessed | 无纪律框架，或流程过重导致无法使用 |

## Skill 类型判断

评估后根据各维度分数分布判断 Skill 类型：

### 重量级（Over-Engineered）
- D1-D5 中有 3+ 个 ≥ 8 分
- D6 ≤ 4 分
- 含义：纪律强但流程过重，简单任务也走完整流水线
- 建议：加强 D6，增加快速模式和复杂度判断

### 轻量级（Lightweight）
- D6 ≥ 7 分
- D1-D5 中有 3+ 个 ≤ 5 分
- 含义：灵活但约束不足，复杂任务可能失控
- 建议：加强 D3（状态门禁）和 D4（TDD）

### 平衡型（Balanced）
- 各维度差异 ≤ 3 分
- 含义：纪律与效率兼顾
- 这是最佳状态

### 偏科型（Lopsided）
- 某一个维度特别高或特别低（与平均分差 > 4 分）
- 含义：有明显的长板或短板
- 建议：优先补短板而非加长板

## 改进优先级定义

### P0 — 安全门缺失（必须立即修复）
- D3 < 4（无基本门禁）
- D4 = 0（完全无测试要求）
- D5 中"受影响文件"和"风险"两项都是 0

### P1 — 效率杀手（强烈建议修复）
- D6 < 4（无快速模式，所有任务走全流程）
- D2 < 4（无文件化，关了就没了）

### P2 — 质量提升（建议修复）
- D1 中 SEARCH 缺失
- D4 中 Red 步骤不强制

### P3 — 锦上添花
- D5 中失败模式定义
- D2 中版本追踪

### P4 — 优化建议
- D6 中自动复杂度判断
- D4 中重构步骤

## 报告输出格式

评估完成后，所有结果写入：

```
.harness/benchmark/<skill-name>/
├── report.md           # 完整评分报告
├── checklist.md        # 审查清单（逐项 pass/partial/fail）
└── improvement.md      # 改进计划（含可直接使用的代码片段）
```
