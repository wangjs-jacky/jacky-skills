# Skill 诊断模式库

> 此文件包含常见的 skill 问题诊断模式，> 在 skill-optimizer 中使用 `grep -E "diagnosis:" references/diagnosis-patterns.md` 快速参考。

## 一、命令未触发问题

### 症状
- skill 文档中定义了命令（如 `/xxx translate`）
- 但执行时命令没有被识别
- 需要显式调用才能执行

### 诊断步骤
1. 检查 skill 的 `<commands>` 部分
2. 磀认命令是否在 commands 中明确定义
3. 检查 `<process>` 中是否有对应的执行逻辑

4. 础认：是独立功能还是流程的一部分

### 优化方向
**方案 A： 添加自动触发**
```markdown
## Phase 4: 研究

### Step 4.4: 自动翻译文档
+ 输出研究发现后，检查是否需要翻译：
+ 如果用户使用中文提问，自动触发翻译。
```

**方案 B: 改为交互提示**
```markdown
## Step 4.4: 沉淀笔记
> 输出研究发现后，询问用户是否需要翻译。
```

**方案 C: 保持手动触发**
- 在 commands 中保留命令
- 在 process 中添加说明
- 不自动触发

## 二、触发条件不匹配
### 症状
- skill 的 description 包含触发词
- 但是实际输入没有被匹配
- 例如：description 包含 "翻译"，但用户输入 "translate"

### 诊断步骤
1. 检查 description 中的触发词
2. 分析触发词的覆盖范围
3. 测试不同输入模式的匹配情况
4. 检查是否有歧义

### 优化方向
**方案 A: 扩展触发词**
```markdown
description: 研究 GitHub 仓库的特定技术实现。触发词：调研下、研究下、学习下、看看 xxx 仓库、分析开源项目、repo-study、翻译、 trans
```

**方案 B: 添加别名**
```markdown
触发条件支持:
- 调研下 xxx`
- `翻译 xxx` (等同于 `/xxx translate`)
```

## 三、功能缺失问题
### 症状
- 预期功能在 skill 中找不到
- 没有对应的 process 步骤
- 没有对应的 commands 定义

### 诊断步骤
1. 明确缺失的功能是什么
2. 检查是否是设计遗漏还是有意为之
3. 评估功能的重要性和通用性
4. 确定实现方式

### 优化方向
**方案 A: 添加新功能模块**
```markdown
## Phase X: 批量操作
### Step X.1: 批量安装
+ 支持批量安装多个 skills
+ ...
```

**方案 B: 添加可选步骤**
```markdown
## Step 4.4: 可选翻译
> 沉淀笔记后，可选：翻译所有文档
```

## 四、自动执行逻辑问题
### 癇状
- 功能应该自动执行但没有
- 需要用户手动触发
- 流程中断

### 诊断步骤
1. 分析 process 的执行顺序
2. 检查是否有条件判断
3. 确认自动执行的触发条件
4. 评估自动执行的风险

### 优化方向
**方案 A: 添加自动触发**
- 在 process 末尾添加自动执行逻辑
- 添加条件判断（如语言检测）
- 提供跳过选项

**方案 B: 改为询问用户**
- 在关键节点询问用户
- 使用 AskUserQuestion
- 提供明确的选项

## 五、上下文管理问题
### 症状
- skill 执行过程中消耗大量上下文
- 主会话上下文不足
- 分析结果不完整

### 诊断步骤
1. 评估 skill 的复杂度
2. 检查是否有大量文件读取
3. 分析是否适合使用 Agent
4. 确认是否需要分阶段执行

### 优化方向
**方案 A: 使用 Agent 模式**
- 将复杂分析委派给 Agent
- 使用 subagent_type=general-purpose
- 在独立上下文中执行

- 返回摘要结果

**方案 B: 分阶段执行**
- 添加 continue 命令
- 支持暂停和恢复
- 使用 task-memory 保存进度

## 六、输出格式问题
### 症状
- skill 输出格式不规范
- 用户难以理解结果
- 后续处理困难

### 诊断步骤
1. 检查输出是否有明确结构
2. 确认是否使用了 markdown 表格
3. 检查是否有代码块格式化
4. 验证链接和引用是否正确

### 优化方向
**方案 A: 标准化输出格式**
- 使用统一的 markdown 模板
- 添加表格总结
- 使用 callout 高亮重要信息
- 添加快速参考部分
