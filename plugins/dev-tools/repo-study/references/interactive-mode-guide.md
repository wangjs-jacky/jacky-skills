## 1. 对话知识实时归档机制

### 核心理念

**对话即知识，有价值的内容应当被自动捕获和沉淀。**

在交互模式的每一步讲解后，自动判断内容价值并归档。

### 归档规则

- **存储路径**：`notes/{主题分类}/{问题关键词}.md`
- **分类方式**：根据问题主题自动创建分类目录
- **执行时机**：每次交互回答结束后，如果判断有价值，立即写入文件
- **内容原则**：提炼知识点，非逐字抄录

### 归档文件模板

```markdown
# {问题关键词}

> 一句话总结

## 问题

{用户的问题}

## 答案

{提炼后的知识点}

## 代码示例（如有）

```{语言}
{代码片段}
```
```

### 归档流程

1. 判断刚才的讲解是否值得归档
2. 如果值得，自动写入 notes/ 目录
3. 简要告知用户已归档（不中断流程）
4. 继续询问是否理解

## 2. 实时思维导图

### 核心理念

**人的记忆最好是一颗树，有一个主干脉络，用户不停的提问就是疯狂往外部探索知识边界。**

在交互模式的整个过程中，实时维护一棵「知识树」，帮助用户建立全局视角，清晰看到自己在知识树的哪个位置。

### 思维导图结构

```
主题（根节点）
├── 核心概念A（主干）
│   ├── 子概念A1（分支）
│   ├── 子概念A2（分支）
│   └── 用户提问延伸 → 新分支A3
├── 核心概念B（主干）
│   ├── 子概念B1（分支）
│   └── 用户提问延伸 → 新分支B2
└── 用户提问 → 新主干C
    └── 子概念C1
```

## 4. 思维导图数据结构

### TypeScript 接口定义

```typescript
interface MindMapNode {
  id: string;                    // 唯一标识
  name: string;                  // 节点名称
  type: 'root' | 'trunk' | 'branch' | 'leaf';  // 节点类型
  status: 'pending' | 'current' | 'completed' | 'exploring';  // 状态
  parentId?: string;             // 父节点ID（根节点无）
  children: string[];            // 子节点ID列表
  content?: string;              // 知识点摘要（完成后填写）
  source?: 'system' | 'user';    // 来源：系统预设 or 用户提问
  createdAt: string;             // 创建时间
  completedAt?: string;          // 完成时间
}

interface MindMap {
  rootId: string;                // 根节点ID
  nodes: Map<string, MindMapNode>;  // 所有节点
  currentNodeId?: string;        // 当前正在讲解的节点
  topic: string;                 // 研究主题
}
```

### 实时更新机制

**1. 初始状态**
- 只有一个根节点（研究主题）
- 状态为 `pending`

**2. 调研完成后**
- 根据核心概念列表，在根节点下创建主干节点
- 所有主干节点状态为 `pending`
- 生成初始思维导图文件

**3. 讲解概念时**
- 将当前概念节点状态设为 `current`
- 在思维导图中标记为「正在讲解」
- 展示当前知识树，让用户看到位置

**4. 讲解完成后**
- 将节点状态设为 `completed`
- 添加知识点摘要到节点 content
- 如果有子概念，创建子节点

**5. 用户提问时**
- 在当前节点下创建新分支节点
- 状态设为 `exploring`
- 标记为「用户提问延伸」

**6. 用户深入追问**
- 在现有节点下添加子分支
- 保持层级关系清晰

**7. 用户跳跃提问**
- 在根节点下创建新的主干分支
- 标记为「用户提问 → 新主干」

## 5. 可视化展示模板

### 每次交互后展示当前知识树

```markdown
📊 **当前知识树**

```
📁 {研究主题}
├── 📂 核心概念A
│   ├── ✅ 子概念A1
│   │   └── 💡 用户追问: 具体如何实现？
│   ├── ✅ 子概念A2
│   └── 🆕 当前讲解: 子概念A3
├── 📂 核心概念B
│   ├── ✅ 子概念B1
│   └── ⏳ 待探索...
└── 🆕 用户提问: {新问题}
    └── ⏳ 待讲解...

**图例说明：**
- 📁 根节点/主题
- 📂 主干概念（核心概念）
- 🌿 分支概念（子概念）
- ✅ 已讲解完成
- 🆕 当前正在讲解/探索
- ⏳ 待讲解/待探索
- 💡 用户深入追问点
- 🔗 用户跳跃提问（新主干）

**进度统计：**
```

```

## 7. 交互流程集成

### Step 5b.1 调研完成后

1. 创建思维导图根节点
2. 根据核心概念列表创建主干节点
3. 生成初始思维导图文件
4. 展示给用户：「这是我们将要探索的知识树」

### Step 5b.2 逐步讲解的每一步

**讲解前：**
```markdown
📊 **知识树导航**

{展示当前知识树，标记即将讲解的节点}

接下来我们将深入: **{概念名称}**
```

**讲解后：**
```markdown
💾 **已归档**: 该概念已保存到 `notes/{主题分类}/{概念名称}.md`
📊 **知识树更新**: ✅ {概念名称} 已完成

{展示更新后的知识树}

---
**Checkpoint**: ✅ 这个概念理解了吗？
...
```

**用户提问时：**
```markdown
💡 **收到提问**: {用户问题}

📊 **知识树生长**: 在 {当前节点} 下延伸新分支

{展示更新后的知识树，标记新增长的分支}

让我为你解答这个问题...
```

### Step 5b.3 总结时

```markdown
🎉 **所有概念讲解完成！**

📊 **最终知识树**:
{展示完整的知识树}

📈 **探索统计**:
- 总概念数: {n}
- 已完成: {n}
- 用户延伸分支: {n}
- 最大探索深度: {n}层
- 知识树文件: `notes/mindmap-{主题}.md`
```

## 用户价值

1. **全局视角**：随时看到自己在知识树的哪个位置
2. **探索路径**：清楚已经学了什么，还有哪些待探索
3. **知识关联**：理解概念之间的层级和关联关系
4. **成就感**：看到知识树逐渐丰满，有学习成就感
5. **复习参考**：思维导图文件可作为后续复习材料

## 归档执行示例

**每次讲解后自动执行：**

1. **判断价值**：根据归档标准判断刚才的讲解是否值得归档
2. **自动归档**：如果值得，立即写入 `notes/{主题分类}/{概念名称}.md`
3. **简要通知**：用一行文字告知用户已归档，不中断教学节奏
4. **继续流程**：立即进入理解确认环节

**归档通知示例：**
```markdown
💾 **已归档**: 该概念已保存到 `notes/workflow/orchestration-pattern.md`
```

## 分支处理详细流程

### 选择 1：理解了

- 更新思维导图：将当前节点标记为 `completed`
- 进入下一个概念
- 如果是最后一个概念，进入总结阶段

### 选择 2：不太理解

- 补充更多解释
- 提供更多代码示例
- 使用类比帮助理解
- **补充内容同样触发归档**：更新已归档文件，追加补充内容
- **思维导图标记**：在当前节点添加 💡 标记表示有深入探讨
- 再次询问是否理解

### 选择 3：有疑问（触发思维导图生长）

- 在思维导图中创建新分支：当前节点下添加「用户提问」子节点
- 状态设为 `exploring`
- 回答用户的具体问题
- 可以引用代码具体位置
- **答疑内容触发归档**：如果答疑有价值，追加到归档文件
- **展示知识树生长**：
  ```markdown
  💡 **收到提问**: {用户问题}

  📊 **知识树生长**: 在 {当前概念} 下延伸新分支

  ```
  📁 {研究主题}
  ├── 📂 {当前概念}
  │   ├── ✅ 已完成内容
  │   └── 🆕 用户提问: {问题摘要}  ← 新生长
  │       └── ⏳ 待探索...
  ```
  ```
- 回答完成后，将新分支标记为 `completed`
- 确认问题解决后继续
